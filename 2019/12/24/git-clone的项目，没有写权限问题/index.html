git clone的项目，没有写权限问题

发表于 2019-12-24

问题
centos系统，git clone到本地的项目，没有写权限

解决办法

  1. 修改项目权限
    chmod -R 777 项目目录
  2. 此时有写权限了，但是git会记录修改的权限信息, 需要忽略掉，否则git pull的时候会冲突
    在项目根目录下执行，git config core.fileMode false

这样就有写权限，并且能正常git pull了
