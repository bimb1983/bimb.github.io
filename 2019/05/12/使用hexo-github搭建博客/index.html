使用hexo+github搭建博客

hexo坏境搭建

  1. 安装并配置node.js坏境，打开cmd，用node -v和npm -v查看是否成功
  2. 安装并配置git坏境，用git –version查看是否成功
  3. 登录Github，新建项目，项目名称格式：账户名.github.io，并且勾选Initialize this repository with a README
    在建好的项目右侧有个settings按钮，点击它，向下拉到GitHub Pages，有个网址，可以通过外网来访问。
  4. 安装Hexo：创建一个blog文件夹，然后通过命令行进入到该文件夹里面。
    4.1 输入npm install hexo -g，开始安装Hexo
    4.2 输入hexo -v，检查hexo是否安装成功
    4.3 输入hexo init，初始化该文件夹, 这里要注意blog文件夹必须是空的，否则会报错。看到“Start blogging with Hexo！” 就成功了！
    4.4 输入npm install，安装所需要的组件
    4.5 输入hexo g，首次体验Hexo
    4.6 输入hexo s，开启服务器，访问该网址，正式体验Hexo

关联github

  1. 将Hexo与Github page联系起来，设置Git的user name和email（如果没有设置过的话）
    git config –global user.name “bimb1983”
    git config –global user.email “xxx@sina.com“
  2. 生成ssh key
  3. 登录github， 点击头像下方的settings，添加ssh key，将id_rsa.pub文件里的内容复制上去
  4. 在本地ssh命令窗口，输入ssh -T git@github.com，再输入yes。如果看到Hi后面是你的用户名，就说明成功了。
  5. 配置Deployment，在blog文件夹中，找到_config.yml文件，修改repo值
    deploy:
    type: git
    repository: git@github.com:bimb1983/bimb1983.github.io.git
    branch: master

发表博客
10 在生成以及部署文章之前，需要安装一个扩展：npm install hexo-deployer-git –save

  1. 在blog路径下打开cmd窗口，执行命令：hexo new post “博客名” ，在blog/source/_posts目录下将会创建一个文件《博客名.md》
  2. 使用命令：hexo d -g，生成以及部署文章
  3. 部署成功后访问你的地址：https://用户名.github.io。会看到生成的文章
